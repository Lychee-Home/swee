# palfeed Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `palfeed_ticker` to swee that polls the `palsave-api` service's `GET /events/new-pals` endpoint every 60s and posts notable pal catches to a dedicated Discord channel.

**Architecture:** Two new swee modules. `swee/palfeed_notability.py` is a pure, dependency-free module (talent scoring, tier classification) importable without a populated `.env`, mirroring how `swee/palworld_settings.py` stays testable today. `swee/palfeed.py` is the integration piece: cursor persistence (`palfeed_state.json`, same pattern as `releases.py`'s `last_release.json`), the `httpx` poll against `palsave-api`, embed formatting, and the `@tasks.loop` ticker itself — wired into `main.py` exactly like `release_ticker`.

**Tech Stack:** Python 3.14, `discord.py` (`discord.ext.tasks`), `httpx` (already a dependency via `swee/config.py`/`swee/rest_client.py`), stdlib `unittest`.

## Global Constraints

- Config vars: `PALFEED_SERVICE_URL` (base URL of the `palsave-api` service; unset/blank disables palfeed entirely) and `PALFEED_CHANNEL_ID` (Discord channel for catch posts) — both optional, same `int(os.environ["X"]) if os.environ.get("X") else None` pattern already used for `RELAY_CHANNEL_ID` in `swee/config.py:10`.
- Poll interval fixed at 60s — no env override (matches `stats_ticker`/`release_ticker` precedent).
- Each tick fetches at most 5 events (`GET /events/new-pals?since=<cursor>&limit=5`) — bounds normal-tick work and throttles catch-up bursts after downtime to 5 posts/tick.
- Cursor (`last_event_id`) persists to `palfeed_state.json` and advances only after an event is fully handled (posted successfully, or skipped as non-notable). On a Discord send failure, stop the batch and leave the cursor where it is — same `break`-on-failure pattern as `release_ticker` in `swee/releases.py:181-192`.
- No decompression, Gvas parsing, or `libooz.so` anywhere in swee — that's entirely `palsave-api`'s responsibility. swee only ever handles small JSON event records.
- `palsave-api`'s event JSON shape (confirmed against the live implementation): `id` (int), `character_id` (str), `level` (int|null), `talent_hp`/`talent_shot`/`talent_defense` (int), `acquisition_type` (`"wild_capture"|"hatched"|"purchased"`), `owner_player_uid` (str|null), `is_rare_pal` (bool), `is_awakening` (bool), `snapshot` (str).
- Run tests with `python -m unittest discover tests -v` from the repo root.

```python
"""Pure notability-scoring functions for the pal-catch recap feed. Zero
swee-internal imports on purpose — this must stay importable without a
populated .env, the same guarantee swee/palworld_settings.py provides.

Talent score (talent_hp + talent_shot + talent_defense, max 300) tiers are
calibrated against one reference save's own distribution, not a verified
community standard — see palsave-api's project memory
(recap_notability_rules.md) for how these were derived. Structural facts
about an event (acquisition_type classification, recruitable-NPC exclusion,
etc.) are palsave-api's responsibility; this module only decides what's
worth posting to a Discord highlight channel.
"""

TALENT_TIERS = (
    (300, "Perfect"),
    (280, "Excellent"),
    (250, "Great"),
)

ACQUISITION_LABELS = {
    "wild_capture": "Caught in the wild",
    "hatched": "Hatched from an egg",
    "purchased": "Purchased from a merchant",
}


def talent_score(event: dict) -> int:
    return event.get("talent_hp", 0) + event.get("talent_shot", 0) + event.get("talent_defense", 0)


def notability_tier(event: dict) -> str:
    if event.get("is_rare_pal"):
        return "Lucky"
    if event.get("is_awakening"):
        return "Awakened"
    score = talent_score(event)
    for threshold, label in TALENT_TIERS:
        if score >= threshold:
            return label
    return ""
```

- [ ] **Step 3: Write swee/palfeed.py**

Create `swee/palfeed.py`:

```python
import json
import logging

import httpx
from discord.ext import tasks

from swee.config import COLOR_PALFEED, PALFEED_CHANNEL_ID, PALFEED_SERVICE_URL
from swee.embeds import broadcast_embed
from swee.palfeed_notability import ACQUISITION_LABELS, notability_tier, talent_score

log = logging.getLogger("swee")

PALFEED_STATE_PATH = "palfeed_state.json"
PALFEED_BATCH_LIMIT = 5

last_event_id = 0  # cached in-memory; mirrors palfeed_state.json on disk


def load_last_event_id():
    global last_event_id
    try:
        with open(PALFEED_STATE_PATH) as f:
            last_event_id = json.load(f).get("last_event_id", 0)
    except FileNotFoundError:
        last_event_id = 0
    except json.JSONDecodeError:
        log.warning("palfeed_state.json is corrupt, starting with no cached cursor")
        last_event_id = 0


def save_last_event_id(event_id):
    global last_event_id
    last_event_id = event_id
    with open(PALFEED_STATE_PATH, "w") as f:
        json.dump({"last_event_id": event_id}, f, indent=2)


async def fetch_new_pal_events(since, limit):
    url = f"{PALFEED_SERVICE_URL}/events/new-pals"
    async with httpx.AsyncClient(timeout=10.0) as client:
        r = await client.get(url, params={"since": since, "limit": limit})
        r.raise_for_status()
        return r.json()


def format_catch_embed(event, tier):
    title = f"{event.get('character_id') or 'Unknown Pal'} — {tier}"
    acquisition = ACQUISITION_LABELS.get(event.get("acquisition_type"), "Acquired")
    level = event.get("level")
    description = acquisition + (f" — Level {level}" if level is not None else "")
    fields = [("Talent Score", f"{talent_score(event)}/300")]
    return title, description, fields


@tasks.loop(seconds=60)
async def palfeed_ticker():
    try:
        events = await fetch_new_pal_events(last_event_id, PALFEED_BATCH_LIMIT)
    except Exception:
        log.exception("palfeed poll failed")
        return

    for event in events:
        tier = notability_tier(event)
        if tier:
            title, description, fields = format_catch_embed(event, tier)
            sent = await broadcast_embed(
                title, description, COLOR_PALFEED, channel_id=PALFEED_CHANNEL_ID, fields=fields,
            )
            if not sent:
                log.warning("palfeed announcement failed for event %s, will retry next tick", event["id"])
                break
        save_last_event_id(event["id"])
```

```python
    guild = discord.Object(id=GUILD_ID)
```
